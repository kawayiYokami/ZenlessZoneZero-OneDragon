# 进入游戏流程说明

## 分层边界

- `OpenGame` 只负责按配置路径启动游戏进程。
- `OpenAndEnterGame` 负责启动游戏、等待窗口就绪、执行 `EnterGame`。
- `EnterGame` 负责登录页识别、账号输入、点击进入游戏、等待进入大世界。

## 首次点击后的分支

`画面识别` 节点只处理第一次 `点击进入游戏` 以及登录页相关状态。第一次点击成功后进入 `进入游戏后操作` 节点，再由 `EnterGame.check_after_first_enter_click()` 统一分流：

- 普通分支：`加载配置数据中` 等待，之后再次出现 `点击进入游戏` 时点击并继续等待，再处理 `登录游戏服务器中`、`登录成功`。`登录成功` 后流转到 `识别加载中` 节点，由框架重试等待 `NOW LOADING` 区域出现；识别到 `加载中` 后进入 `加载中` 节点，直到该区域消失后再流转到 `识别弹窗和大世界`。如果 `识别加载中` 重试后仍未命中，也会兜底进入 `识别弹窗和大世界`。
- 更新分支：`资源下载中` 最多等待 10 分钟；更新完成后识别 `游戏数据已更新` 弹窗并点击 `确定`。
- `确定` 后游戏会回到第一次点击进入游戏之前，因此 `进入游戏后操作` 返回 `游戏数据已更新`，流转回 `画面识别` 重新走标题页和第一次点击。
- 登录失败不进入主分支动作表。`画面识别` 会沿用现有 `match_login_error()` 处理已经覆盖的 `确定`、`重试` 弹窗；进入 `进入游戏后操作` 后，则只有脚本已经完成第二次 `点击进入游戏`，或识别到 `登录游戏服务器中`、`登录成功` 之后，才会再调用这个兜底，未覆盖的失败后续单独补。

这些文本动作通过 `round_by_ocr_and_click_with_action()` 内联维护在 `EnterGame.check_after_first_enter_click()`，对应 OCR 区域是 `assets/game_data/screen_info/enter_game.yml` 中的 `进入游戏点击后状态`。加载节点使用 `assets/game_data/screen_info/loading.yml` 中的 `加载中` 区域，更新完成提示使用 `游戏数据更新提示` 和 `游戏数据更新-确定` 两个区域，不要在 Python 里硬编码坐标。
