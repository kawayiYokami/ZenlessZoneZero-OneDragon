# 自动战斗补充说明

## 队伍角色识别兜底

自动战斗中的队伍角色识别，原本会在连续多次识别结果一致后，停止全量匹配，转为基于当前队伍信息的低成本识别。

当游戏内出现同一队伍在战斗中循环切换的新模式时，旧的低成本识别可能会因为候选角色被固定，导致当前画面一个角色都识别不到。

针对这个场景，当前实现增加了一个兜底策略：

- 当普通攻击按钮可用，说明当前仍处于战斗画面。
- 如果这一帧角色识别结果为“一个角色都识别不到”，则不立即常驻开启全量识别。
- 仅将下一次角色识别强制切回“全量匹配所有角色”，并重置识别节流时间。

这样可以在兼容新模式的同时，尽量减少常规战斗中的模板匹配压力。

## 战斗按钮提示

战斗画面中的部分按钮会通过模板匹配写入状态，供自动战斗状态判断与调试提示使用。

- 普通攻击按钮用于快速判断当前是否仍处于战斗画面。
- 切换后援按钮每 1 秒检测一次，识别命中时会写入状态 `按键可用-切换后援`。
- 自动战斗可直接执行原子操作 `按键-切换后援`，默认键盘键位为 `R`。
